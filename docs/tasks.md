## Техническое решение
- Frontend: `Next.js + TypeScript + Tailwind CSS`
- Рендеринг: статические страницы по умолчанию, серверная логика только где нужна
- Деплой сайта: `GitHub Pages`
- Медиа: `Cloud.ru Object Storage` + `CDN`
- Медиа fallback без новых расходов: `wedding-vps`
- `Cloud.ru VPS`: резерв для будущего backend, не обязательная часть текущего сайта
- Лиды: по необходимости через отдельный backend в существующую админку и Telegram
- Формы: собственная обработка, без зависимости от Tilda
- Источник старта: Tilda static export в текущем репозитории
- Видео: MP4 как базовый формат, poster + lazy load, при необходимости HLS
- Видео QA: сначала тестовый файл, потом массовый перенос
- Плеер: нативный HTML5 video, без обязательного стороннего плеера
- Видеоблоки и любые внешние видеоисточники подключать только после восстановления всего лендинга без видео.

## Примечания по рискам
- Не тащить Boomstream как единственный источник видео.
- Не рассчитывать, что экспорт Tilda даст готовый backend.
- Не добавлять лишнюю архитектуру до того, как станет понятен финальный объем форм и интерактивности.
- Не путать Cloud.ru "Проект" в меню с реальной виртуальной машиной.
- Не удалять и не пересоздавать `wedding-vps` без отдельной команды.
